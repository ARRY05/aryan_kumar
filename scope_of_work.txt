What I need:
The application should be a responsive full stack portal with these main areas in order: Home Page, Game Playback View, User Dashboard, and Admin Panel.

All token or license inquiries happen through direct chat integration. No payment gateway, no checkout system.

1. Home Page and Grid Layout
* Brand name (Romlex) and search bar at the top
* Hero slider for featured games
* Category sidebar (Action, Puzzle, Strategy, etc.)
* Main grid showing game cards
* Layout: 4 to 5 cards per row on desktop, 2 per row on tablet, 1 per row on mobile
* Game card hover effects: CSS lift and drop shadow

2. Game Playback View
* Dedicated page that embeds the selected game in an iframe
* Sidebar displaying publisher info, player ratings, and view count
* Add to Favorites button
* Inquire for License via chat button

3. User Dashboard
* Simple user registration and login
* History tab showing recently played games
* Favorites tab showing saved games
* Profile section to update username and avatar

4. Admin Panel
* Secure login restricted to site owners
* Dashboard form to add new games (title, category, iframe URL, and WebP thumbnail upload)
* List view to edit existing games or delete them
* Toggle switch to make a game active or hidden on the frontend

5. Chat Integration (Only Inquiry Method)
* Clicking the Inquire for License button opens a direct chat link (like wa.me)
* The link must contain a pre-filled message stating the exact game title and inquiry type
* Desktop fallback modal with contact info if the chat application is not installed

Animations:
All animations should be lightweight and CSS based.

Allowed tools:
* Pure CSS animations and transitions
* AOS library for fade in effects

Required animations:
* Game card hover lift and drop shadow
* Hero slider CSS transitions
* Sections fade in on scroll

Accessibility:
* Respect prefers-reduced-motion CSS media query

Database and Architecture:
* Relational database using SQLite or PostgreSQL
* Backend built with Python (FastAPI or Flask preferred)
* Frontend built with HTML, CSS (Tailwind or Bootstrap allowed), and vanilla JavaScript
* The system reads and writes to the database dynamically for all game listings

Tech Stack and Quality:
* Mobile first responsive design
* Lighthouse performance score of at least 85
* Tested on Chrome, Safari, Firefox, Edge
* All images optimized as WebP, max 200KB per image, lazy loaded

SEO Basics:
* Page title and meta description
* Open Graph tags
* Sitemap.xml and robots.txt
* Semantic HTML

Accessibility Basics:
* Alt text on every image
* ARIA labels on icon only buttons
* Keyboard navigation

Docker Setup:
* Dockerfile based on Alpine (Nginx for static files, plus Python environment for backend)
* docker-compose.yml
* One command startup: docker compose up
* Default port 8080, README explains how to change it

3. Dockerfile and docker-compose.yml
